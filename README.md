# 坦克大战 · 无尽版

一款基于 TypeScript、Canvas 和 Vite 构建的全屏无尽坦克战斗游戏。玩家需要在逐步扩大的战场中保护基地、清理敌军、收集并升级武器，同时应对不断变化的敌人编成、地形和关卡事件。

## 核心特性

- 全屏战斗界面，HUD 集中显示关卡、分数、击破数、剩余敌军、生命、基地耐久和武器能量。
- 地图每 5 关扩大一次；相机会在玩家离开安全区后平滑跟随，并通过小地图显示视野位置。
- 14 种炮弹，共享可恢复武器能量；重复拾取同种武器可升级到 Lv5。
- 7 类常规敌人和 1 类 Boss，每类常规敌人都有 3 档升级外形与属性成长。
- 动态难度、Boss 关、随机事件、持续增益和三类地形模板。
- 基地拥有成长耐久、紧急维修和维修护盾，不再因为一次失守立即结束整局。
- 支持单人无尽、双人合作无尽和最多 10 人生存战。
- 联机模式支持快速匹配、创建房间、房间码加入和房主自由开局。
- 支持断点续玩，以及服务器统一维护的单人榜、双人榜和十人榜。

## 游戏模式

### 单人无尽模式

现有本地无尽玩法。玩家独自守卫基地，收集并升级武器，挑战不断扩大的地图、随机事件和 Boss 关卡。支持本地断点续玩。

### 双人无尽模式

两名玩家合作守卫同一基地，共享 6 条初始装甲储备和关卡进度。友军之间不会互相造成伤害，敌军会从存活玩家中选择目标。基地失守或玩家阵亡会消耗一条共享储备；储备尚未耗尽时，基地会紧急维修、阵亡玩家也可重新加入战斗。每关结束后由房主从三种武器中选择团队奖励，30 秒未选择则由服务器随机决定。

### 10 人生存模式

最多 10 名玩家自由对战，每名玩家拥有独立阵营和 5 条初始生命。战场会分阶段缩小安全区，安全区外持续受到伤害；地图中会刷新武器和少量额外生命。生命耗尽后玩家被淘汰，最后一名生存者获胜。

双人无尽和 10 人生存都支持：

- 快速匹配：双人模式凑满 2 人、生存模式凑满 10 人后自动开局。
- 创建房间：生成 6 位房间码；自定义房间至少 2 人即可由房主手动开始。
- 房间码加入：输入房间码加入指定队伍。
- 服务器权威模拟：以 60 Hz 处理移动、碰撞、伤害、AI、炮弹效果和计分，并广播压缩快照。
- 两种联机模式均复用完整的 14 种武器规则，武器通过关卡奖励或场内资源逐步解锁。

## 武器与能量

玩家拥有 100 点共享武器能量，每秒恢复 18 点。普通弹无限使用，特殊武器按次消耗能量；能量不足时武器仍保持选中，恢复后可继续发射。重复获得同一种炮弹会提升等级，并强化伤害、冷却、能量消耗或对应特殊效果。

同阵营炮弹不会互相抵消。敌我双方的同种炮弹相撞时按照实际伤害判断：伤害相同则同时消失，高伤害炮弹会抵消低伤害炮弹并继续飞行。

| 炮弹 | 能量 | 主要效果 |
| --- | ---: | --- |
| 普通弹 | 0 | 标准单发，无限使用 |
| 速射弹 | 5 | 极短冷却，适合持续压制 |
| 重炮弹 | 24 | 3 点基础伤害，可击破钢板 |
| 穿甲弹 | 20 | 贯穿多个敌人 |
| 爆破弹 | 28 | 命中后产生范围爆炸 |
| 散射弹 | 18 | 扇形多弹丸覆盖 |
| 追踪弹 | 26 | 自动追踪最近敌人 |
| 弹跳弹 | 18 | 在砖墙、钢板和地图边界间反弹 |
| 冰冻弹 | 24 | 范围减速与冻结 |
| 聚焦激光 | 26 | 高速、短冷却、直线贯穿 |
| 灼热等离子 | 30 | 范围爆炸并持续灼烧 |
| 连锁电弧 | 32 | 追踪命中后连锁附近目标 |
| 震荡冲击弹 | 30 | 大范围击退敌人并打乱阵型 |
| 磁轨歼灭弹 | 46 | 高伤害、多重贯穿、击退并破坏钢板 |

不同炮弹拥有独立的弹体造型、轨迹、命中特效和合成音效。燃烧、冻结、追踪、反弹、连锁、击退、穿透和钢板破坏均由实际战斗逻辑处理。

## 敌人体系

| 敌人 | 武器 | 战斗特性 |
| --- | --- | --- |
| 游骑侦察车 | 普通弹 | 高速机动，主动追击玩家 |
| 双联压制车 | 速射弹 | 高频火力压制 |
| 堡垒重装车 | 重炮弹 | 高耐久、有装甲，倾向攻击基地 |
| 长枪狙击车 | 穿甲弹 | 精准远程射击 |
| 侧袭弹跳车 | 弹跳弹 | 高机动和高闪避，利用墙体反弹攻击 |
| 攻城爆破车 | 爆破弹 | 范围破坏，优先威胁基地 |
| 猎手指挥车 | 追踪弹 | 高价值、高掉落率，持续追猎玩家 |
| 多炮塔战争堡垒 | 重炮弹幕 | 每 5 关出现，拥有多炮塔齐射和斜向弹幕 |

常规敌人会随难度档位获得 3 档外形变化，同时提高生命、速度、武器等级和辨识度。高关卡会逐渐加入更复杂的敌人编成。

## 基地规则

- 基地初始拥有 5 点耐久，随后缓慢成长，最高 8 点。
- 单次敌方炮弹命中只扣除 1 点基地耐久，范围爆炸覆盖基地时也只结算一次。
- 玩家炮弹会被基地阻挡，但不会伤害己方基地。
- 基地耐久归零时，如果仍有备用生命，会消耗一条生命并恢复一半基地耐久。
- 紧急维修后基地获得 5 秒无敌护盾；只有基地失守且没有备用生命时才会结束游戏。

## 关卡与难度

- 地图尺寸和敌军强度每 5 关提升一个档位。
- 每 5 关出现一次 Boss，击破 Boss 后才可进入下一关。
- 地形在开阔战、走廊战和要塞战之间轮换，并保证出生点与基地之间可通行。
- 战争迷雾、湿滑地面等随机事件会改变单局战斗节奏。
- 连续无伤通关会提高压力档位，连续受挫会降低压力档位。
- HUD 的“剩余敌军”包含当前存活、等待生成以及尚未登场的 Boss。

## 本地运行

环境要求：Node.js 20.19+ 或 22.12+，推荐使用 pnpm。

安装依赖：

```powershell
pnpm install --frozen-lockfile
```

完整体验需要分别启动后端和 Vite 开发服务器：

```powershell
# 终端 1：排行榜 HTTP API 与 WebSocket 联机服务
pnpm run server

# 终端 2：前端开发服务器
pnpm dev
```

后端默认监听 `8787` 端口，同时提供 `/api` 排行榜接口和 `/ws` 联机服务。只体验单人核心玩法时可以仅启动 Vite，但排行榜和联机功能需要后端。

生产构建与测试：

```powershell
pnpm build
pnpm test
```

其他脚本：

| 命令 | 用途 |
| --- | --- |
| `pnpm dev` | 启动 Vite 开发服务器 |
| `pnpm build` | 检查客户端与服务器 TypeScript，并生成生产包 |
| `pnpm typecheck:server` | 单独检查服务器和联机模拟代码 |
| `pnpm preview` | 预览生产构建 |
| `pnpm test` | 运行全部 Vitest 测试 |
| `pnpm test:watch` | 监听模式运行测试 |
| `pnpm coverage` | 生成测试覆盖率 |
| `pnpm latency:multiplayer -- <ws-url> [samples]` | 测量 WebSocket 建连与 RTT，默认采样 10 次 |
| `pnpm run server` | 启动排行榜与联机后端，默认端口 8787 |

## 联机配置

开发环境默认连接 `ws://当前主机:8787/ws`；生产构建默认使用网页同源的 `/ws`，可自动适配 HTTPS/WSS 和反向隧道。使用其他联机服务器时，在启动 Vite 前设置：

```powershell
$env:VITE_MULTIPLAYER_URL='ws://127.0.0.1:8787/ws'
pnpm dev
```

可用独立脚本测量任意 WebSocket 链路的建连、welcome、原生 RTT 和应用层 RTT：

```powershell
pnpm latency:multiplayer -- wss://your-host.example/ws
```

## 排行榜配置

排行榜始终使用服务器数据，默认请求同源 `/api`。开发环境连接独立后端时可覆盖地址：

```powershell
$env:VITE_LEADERBOARD_URL='http://localhost:8787/api'
pnpm dev
```

后端支持以下环境变量：

| 环境变量 | 默认值 | 用途 |
| --- | --- | --- |
| `HOST` | `0.0.0.0` | HTTP 与 WebSocket 监听地址 |
| `PORT` | `8787` | HTTP 与 WebSocket 监听端口 |
| `LB_DATA` | `server/data/leaderboard.json` | 排行榜 JSON 数据文件；默认目录已被 Git 忽略 |
| `LB_ALLOW_CLEAR` | 未设置 | 设为 `1` 时开放 `POST /api/clear`，生产环境建议保持关闭 |

单人成绩通过 HTTP 提交，并执行分数、击杀、关卡合理性校验和签名校验；双人和十人模式由权威联机服务器在结算时直接写榜。

## 生产部署

`deploy/` 提供一套 Linux 部署模板：

- `tankwar.service`：以 `www-data` 运行后端，工作目录为 `/opt/tankwar`，排行榜写入 `/var/lib/tankwar/leaderboard.json`。
- `nginx.conf`：从 `/opt/tankwar/dist` 提供前端静态文件，并将 `/api/` 与 `/ws` 反向代理到 `127.0.0.1:8787`。
- `cloudflared.yml`：Cloudflare Tunnel 示例配置。部署前必须在私有副本中替换 `YOUR_TUNNEL_UUID` 和 `tankwar.example.com`。

生产构建：

```bash
pnpm install --frozen-lockfile
pnpm build
```

不要把真实 Tunnel UUID、生产域名、源站 IP 或 `.cloudflared` 凭据 JSON 提交到仓库。项目已经忽略 `.cloudflared/`、`deploy/*.local.*` 和常见凭据文件；生产值应放在服务器上的私有配置中。
